# Web3 Documentation Audits

A portfolio of documentation audits for open-source blockchain and DeFi protocols.

Each audit evaluates a project's documentation against industry standards across five categories: **Completeness**, **Accuracy**, **Discoverability**, **Maintenance**, and **Security Documentation**. Findings are rated by severity (Critical, High, Medium, Low) based on developer and user impact.

## Audit Method

Each audit follows a structured process:

1. **Standard Files Check** — This is the verification of the presence and quality of the README.md, CONTRIBUTING.md, SECURITY.md, CODE_OF_CONDUCT.md, CHANGELOG.md, and LICENSE.

2. **Accuracy Verification** — Cross-reference of documentation claims against the actual state of the project (deployed contracts, live features, current team, active links).

3. **Cross-Reference Analysis** — Evaluating the connections between repo docs, live documentation sites, and community resources.

4. **Security Documentation Review** — Assessing vulnerability disclosure policies, audit report accessibility, bug bounty information, and supported version tables.

5. **Community Signal Check** — Reviewing open issues and community channels for patterns that cause user confusion to help indicate documentation gaps.

Scoring is based on the Linux Foundation's Documentation Health Index and Write the Docs community standards.
